# PersonaLens
一个把人放进电脑的孪生数字系统，是一个基于证据链的人物背景建模与行为推演实验项目。
Evidence-based persona modeling and scenario behavior inference engine powered by LLM memory and temporal knowledge graphs.
我们经常会遇到一个问题，就是如何和喜欢的人进行沟通。

害怕出错，害怕她不喜欢，害怕自己没做好。。。。

其中，勇气非常重要，但是很多人都下不了勇气去表达，很多人会把这个归咎于自己的经历（一说话就脸红，家庭不好，经济实力不够....)

当然，我相信一个人的性格是由于其本身的经历而塑造的，正因如此，那么，如果我们充分或者接近充分的了解一个人过往的经历和平时的生活方式，喜欢吃什么，喜欢看什么电视剧，喜欢打什么游戏，我们就可以对这个人进行建模。

其实也不完全，最近在读一本书叫做被[讨厌的勇气]：一个人也可以被自己困住，过往的经历只是限制自己的借口罢了。

一般能打胜仗优秀将军都不是名将，名将像赵云，关羽都是以少胜多而出名，但是优秀的将军是不会把自己放在劣势当中的，那么找对象也是这样子的，一定要做好准备。

通过这个人的背景资料，成长经历，公开表达（比如说qq空间，微博账号，微信朋友圈），聊天习惯，我们就能够找到crush心中的锚点。

虽然这个想法还是比较危险的，因为可能设计社会工程学，操控他人，侵犯他人隐私，所以这个项目全部的资料都是得到授权的。

本项目旨在探索如何将一个人的背景资料、成长经历、公开表达、互动记录和用户提供的上下文信息，转化为结构化记忆、价值观线索、情绪触发点、决策模式和情景行为假设。

与传统用户画像系统只分析兴趣偏好不同，PersonaLens 更关注深层人物理解。系统不仅尝试总结一个人喜欢什么，还希望进一步分析：为什么某些事情对这个人重要，过去经历如何影响其价值观，不同情境下可能出现怎样的情绪模式，以及在未来场景中可能做出怎样的选择。

系统采用谨慎的证据链工作流：

1. 输入人物背景资料
2. 提取事件、情绪、主题、价值观、边界和决策线索
3. 将资料存入本地结构化记忆数据库
4. 构建时间化个人知识图谱
5. 生成带证据链的人物假设
6. 在未来情景中推演可能反应
7. 允许用户修正、拒绝或更新系统判断

PersonaLens 不声称能够真正看透一个人的内心，也不提供心理诊断、操控策略或绝对预测。系统输出的是基于已有资料的、可解释的、带不确定性的、可修正的人物理解假设。

本项目适用于人本 AI 研究、个人背景建模、长期记忆系统、情景推演、关系理解、沟通分析、虚构角色建模和数字孪生实验。

## 核心原则

* 证据优先
* 数据来源应基于授权或用户主动提供
* 优先本地存储
* 不进行未授权数据采集
* 不用于操控、胁迫或诱导
* 区分事实与假设
* 每个推断都应有证据支撑
* 每个假设都应可编辑、可拒绝、可修正

## 示例

输入：

> 她经常说自己不喜欢被催，也不喜欢别人替她安排事情。压力大的时候，她会减少沟通，更希望按自己的节奏处理问题。

输出：

* 可能的价值观线索：自主权
* 可能的情绪触发点：被催促、被安排
* 可能的沟通偏好：低压力支持
* 可能的未来反应：在压力下被频繁追问时，可能减少表达或回避沟通
* 置信度：中等
* 证据来源：多次表达“不喜欢被催”和“不喜欢被安排”
* 建议验证方式：通过开放式问题确认，而不是直接下结论

PersonaLens 不是“读心工具”。
它是一个将背景资料转化为结构化、可解释、可修正人物理解的实验系统。
